some things you may have heard about jQuery...

But seriously...

  • jQuery solves many of the traditional problems involved in developing JavaScript intensive web applications

jQuery facts

  • Most popular JavaScript libary in use today
  • Used in 46% of top 10,000 websites
  • Embedded in many platforms (.NET, Grails)

So what is jQuery's magic formula?

Selectors


    $('#myDiv') // id selector
    $('.myClass')  // class selctor
    $('div')  // type selector

Selectors


    $(':input') // special selectors
    $('#main > div')  // child selctor
    $('#main form')  // descendant selector

DOM Manipulation


    $('myDiv').addClass('myClass')
    $('myDiv').removeClass('myClass')

DOM Manipulation


    $('myDiv').show()
    $('myDiv').hide()

Event handling


    $(':button').click(function() {
        this.attr('disabled', 'disabled');
    });

Ajax support - Get data

    $.get('test.html', function(data) {
      $('.result').html(data.title);
    });

Ajax support - load content


    // result of request will replace 
    // content of element with id result
    $('#result').load('test')

Let's look at some plugins

Some criteria

  • Well documented
  • Flexible design for many requirements
  • Friendly (non-GPL license)
  • Widely used
  • Actively maintained

jQuery Validator

  • http://bassistance.de/jquery-plugins/jquery-plugin-validation/
  • Client side validation

Grails has validation but...

  • Validation requires a round trip
  • Messages appear at the top (not user friendly)
  • Default messages are kind of ugly

jQuery validator makes validation easy

  • Make a selector for your form and then call .validate()
  • Add classes with rules (required,email,url, etc)
  • Or attach the rules via JavaScript

Flexibility

  • Specify when validation fires - keystroke, unfocus, submit
  • Fine control over where validtion messages appear
  • Internationalization support

Demo

DataTables

  • http://datatables.net/
  • Rich dynamic table builder

DataTables features

  • Wraps html table element with dynamic paging, sorting, searching
  • Lots of flexibility with table look and feel
  • JQuery UI support

DataTables performance

  • Easiest to do client side - load the whole table up front
  • But for large datasets has a server paging API
  • Lots of loading options available

Table alternatives

  • Many other solutions in this sphere
  • jqGrid - more complex api
  • Ingrid - documentation lacking
  • Flexigrid - not as feature rich
  • Grid widget planned for jQuery UI 2.1 - looks promising

Full Calendar

  • http://arshaw.com/fullcalendar/
  • Dynamic calendars (think Google Calendar)

Full Calendar features

  • Load event data via JSON/Javascript
  • Rich and elegant eventing model (drag & drop, click events)
  • Simply mark select a div and call .fullCalendar()

Full Calendar UI

  • Month, week and day views
  • Fine control over rendering options
  • jQuery UI support

dynatree

  • http://code.google.com/p/dynatree/
  • Used for builing rich tree hierarchies

dynatree features

  • Init through Javascript/JSON or HTML
    • Supports full loading or lazy load through Ajax
    • Support for icons and checkboxes
    • Rich eventing model

BBQ

  • http://benalman.com/projects/jquery-bbq-plugin/
  • Bookmark/history support

BBQ functionality

  • When the hash value of a U
  • Uses HTML5 hashchange event or adds this feature for older browsers
  • Treat the hash the same as a request string
  • Ex. #tab=classes&partner=2
  • Retrieve individual params with deparam()
  • Change or add with pushState()

Warning

  • Uses an iframe with IE7 and earlier
  • Stuck on 1.3pre release for over a year
  • Other alternatives may be a better choice - search on Single Page Applications
